  <action>
    **1a — Skip-to-content link in Layout.astro:**
    In `src/layouts/Layout.astro`, insert an `<a>` tag as the **first child of `<body>`** (before `<slot />`). Use this exact markup:

    ```astro
    <a
      href="#main-content"
      class="sr-only focus:not-sr-only focus:fixed focus:top-4 focus:left-4 focus:z-50
             focus:px-4 focus:py-2 focus:bg-[#4a90d9] focus:text-[#0f0f0f] focus:rounded
             focus:outline-none"
    >
      Skip to content
    </a>
    ```
    - `sr-only` hides it visually by default
    - `focus:not-sr-only` makes it appear when focused via keyboard (Tab)
    - `focus:fixed focus:top-4 focus:left-4 focus:z-50` positions it in the top-left corner
    - The accent blue background with dark text ensures high contrast when visible
    - No JavaScript needed — this is a pure HTML anchor pattern (WCAG Technique G1)

    **1b — Add `id="main-content"` to each component's `<main>` element:**
    In each of these three component files, add `id="main-content"` to the opening `<main>` tag:

    - `src/components/Home.astro`: Change `<main class="flex-auto mx-auto max-w-[720px] w-full px-4 py-8">` to `<main id="main-content" class="flex-auto mx-auto max-w-[720px] w-full px-4 py-8">`
    - `src/components/About.astro`: Same change — add `id="main-content"` to the `<main>` tag
    - `src/components/Projects.astro`: Same change — add `id="main-content"` to the `<main>` tag

    **1c — Fix external link security in Footer.astro:**
    In `src/components/Footer.astro`, both external links have `target="_blank"` without `rel="noopener noreferrer"`. Fix both:

    - The "Powered by Astro" link: Add `rel="noopener noreferrer"` alongside existing `target="_blank"`.
    - The "hosted on GitHub Pages" link: Add `rel="noopener noreferrer"` alongside existing `target="_blank"`.

    Current state (both links need fixing):
    ```astro
    <a target="_blank" href="https://astro.build/" class="font-bold underline hover:text-[#6bb3f0]">Astro</a>
    <a target="_blank" href="https://pages.github.com/" class="font-bold underline hover:text-[#6bb3f0]">GitHub Pages</a>
    ```

    After fix, both should include `rel="noopener noreferrer"`:
    ```astro
    <a target="_blank" rel="noopener noreferrer" href="https://astro.build/" class="font-bold underline hover:text-[#6bb3f0]">Astro</a>
    <a target="_blank" rel="noopener noreferrer" href="https://pages.github.com/" class="font-bold underline hover:text-[#6bb3f0]">GitHub Pages</a>
    ```

    This prevents the opened page from accessing `window.opener` (tab-napping security vulnerability) and prevents referrer leakage.
  </action>
